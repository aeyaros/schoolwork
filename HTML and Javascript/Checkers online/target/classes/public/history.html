Version History / Release Notes
Current version: 0.7β

Fixes:
  • Fixed illegal move highlighting.
  • Joining a random game now works properly.
  • Additional event handlers added for clicking to allow for more responsiveness.
Known Issues:
  • Crowning is fixed, although there is a bug where crowns may change color.

0.6α

Known issues:
  • Server doesn't prevent game from connecting to itself; when starting a random game it immediately connects to itself rather than waiting for another instance of the client to ask for a random game.
  • Clients aren't yet prompted to join games; both clients must try to connect with each other. This requires a first client to make an initial request, a second to make a request (after which the second will enter the game), and another request for the first client to join the already open game.
  • Some suggested moves may be incorrectly highlighted. However, they will be rejected by the server if attempted.
  • Pieces are not crowned; the server doesn't send back a representation of them as kings; instead it keeps the pawn representations, and prevents them from being moved.
  • Sometimes, a user must click twice on the destination square for their current move.
  • Sometimes, highlighted moves may be displayed more than once when the user is clicking around, showing a stronger highlight color (i.e. drawing multiple translucent objects on top of each other on the HTML5 canvas)
  • Error-flash events (where board briefly flashes light) occurs too often
